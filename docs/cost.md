## Cost model

**MotherDuck backend.** `compute_seconds` comes from
`MD_INFORMATION_SCHEMA.QUERY_HISTORY`; `storage_bytes` from
`MD_INFORMATION_SCHEMA.STORAGE_INFO_HISTORY` (Business plan gate — the
asset falls back to a single zero-row if the views are unreachable).
`motherduck_cost_usd` is `compute_seconds *
MOTHERDUCK_COST_PER_COMPUTE_SECOND` with the rate hardcoded in
`packages/databox/databox/orchestration/domains/analytics.py` at
MotherDuck's current public "Standard" price. **Update that constant
when MotherDuck's price changes** — there is no pricing API to pull
from.

**Local backend.** `storage_bytes` is the sum of `os.stat(...).st_size`
for every `data/*.duckdb` file. `compute_seconds` and `query_count` are
zeroed out; `motherduck_cost_usd` is NULL. Local is essentially free —
this page exists so forkers can still see disk-growth trends.

## Extending alerts

The ticket scope does not include budget alerts. If you want one, pair
the `freshness_violation_sensor` pattern from `docs/freshness.md` with a
threshold on the freshest `motherduck_cost_usd` row. A minimal sketch
lives under
[`docs/freshness.md#wiring-to-an-alert-channel`](freshness.md#wiring-to-an-alert-channel).
