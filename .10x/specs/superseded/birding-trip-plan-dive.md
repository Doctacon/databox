# Birding Trip Plan MotherDuck Dive

Superseded by `.10x/specs/local-birding-trip-copilot-app.md` after the user selected a local React product and removed MotherDuck from the repository.

## Purpose and scope

This spec governs the MotherDuck Dive surface for the Birding Trip Copilot.

The Dive is the first portfolio-facing app surface. It MUST visualize persisted trip-plan artifacts generated by the Python/Google ADK planner; it is not the agent runtime itself.

## Behavioral contract

The Dive MUST read trip-plan data from MotherDuck/DuckDB SQL artifacts produced by the planner workflow. It MUST NOT require browser-side secrets or direct calls to private API tokens.

The Dive SHOULD let a viewer inspect:

- the selected/latest trip plan,
- plan summary and field-ready recommendation,
- high-likelihood species,
- uncommon-but-plausible target species,
- weather/elevation context,
- evidence/provenance behind recommendations,
- media/call examples and license/attribution metadata where available,
- agent/tool trace summary sufficient to show the workflow was agentic.

The Dive MAY include filters or selectors for trip plan, species confidence group, evidence source, or target category if the persisted data supports them.

## Architecture

- Python/Google ADK generates and persists trip plans.
- The Dive uses SQL queries over persisted tables/views.
- Heavy data shaping belongs in SQLMesh/planner-ready views, not React.
- React handles layout, interactivity, loading states, empty states, and errors.
- The first version SHOULD be local-previewable before saving/publishing to MotherDuck.

## UX expectations

The Dive SHOULD tell one clear story:

> "Here is the recommended birding outing, why these birds are likely or worth targeting, and what evidence supports the plan."

A first version SHOULD include:

1. Header/summary card for location, date/time, duration, and weather.
2. Field plan narrative.
3. Ranked species table/cards with likelihood category and rationale.
4. Evidence/detail table showing source, recency, provenance, and caveats.
5. Tool trace or workflow summary section.

## Acceptance criteria

- The Dive can render from persisted trip-plan SQL artifacts without running the Python agent in-browser.
- The Dive shows the final field plan and recommendation sections from `.10x/specs/birding-trip-copilot.md`.
- The Dive exposes evidence/provenance, not just the final prose.
- The Dive handles loading, empty, and error states.
- The Dive does not include browser-side API secrets.
- The implementation documents how to preview/update/save the Dive.
