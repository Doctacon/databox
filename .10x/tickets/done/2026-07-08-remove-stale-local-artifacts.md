# Remove stale local/generated artifacts

## Scope

Remove files and directories that are objectively local/generated artifacts from prior Databox iterations and update cleanup/ignore rules so they do not keep reappearing.

In scope:

- Python/test/tool caches: `.coverage`, `.pytest_cache/`, `.mypy_cache/`, `.ruff_cache/`, project `__pycache__/`, project `*.egg-info/`.
- Stale SQLMesh cache/output under `transforms/main/.cache/` and accidental local data under `transforms/main/data/`.
- Accidental package-local dlt data under `packages/databox/data/`.
- Regenerable docs site build under `site/`.
- Regenerable `.dive-preview/node_modules/` and local `.dive-preview/.env`.
- Empty untracked state/scaffold directories: `.dagster_state/`, `.dlt_state/`, `.duckdb_state/`, `.sqlmesh_state/`, `.worktrees/`, `docs/images/`, `transforms/_shared/`.
- Root `logs/` generated by earlier SQLMesh invocations.
- `.gitignore` and `Taskfile.yaml` cleanup coverage for the above generated artifacts.

## Explicit exclusions

- Do not delete `.venv/`; it is the active local environment.
- Do not delete `.env`; it may contain local secrets/configuration.
- Do not delete `data/`; it contains the current local DuckDB/dlt state.
- Do not delete `.dagster/` while `task dagster:dev` is running.
- Do not delete `.logs/`; `.10x/evidence/2026-07-07-cdm-sqlmesh-migration.md` references the current full-refresh log there.
- Do not delete `.schema/`, `.10x/`, `.pi/skills/`, source code, tests, docs source, Soda contracts, or SQLMesh source models.

## Acceptance criteria

- In-scope generated artifacts are removed.
- `.gitignore` ignores recurring generated artifact locations not already covered.
- `task clean` removes the safe generated artifacts without dropping `.venv`, `.env`, `data/`, `.dagster/`, or `.logs/`.
- `git clean -nd` has no untracked empty/generated directories from this cleanup scope.
- `git clean -ndX` no longer reports the cleaned safe artifact directories except intentionally preserved `.venv`, `.env`, `data/`, `.dagster/`, and `.logs/`.
- `task ci` passes after cleanup rule changes.
