# Public README: details on demand

The root README positions Databox first as a local-first DuckDB data warehouse.
Rufous is a reference consumer, not the repository's primary identity.

Keep the README to the shortest useful path: purpose, one-line data flow, the
post-ingestion modeling-skill chain, quickstart, and links to deeper
documentation. The `annotate-sources` → `create-ontology` → `generate-cdm` →
`create-transformation` workflow is core differentiation, not optional detail;
present its taxonomy, ontology, CDM, and SQLMesh outputs compactly.

Broader architecture explanations, capability inventories, operational
guidance, repository layout, and product feature detail belong in `docs/` and
should be linked only when they help a new reader take the next step.
